## 🟥 우선순위 높음 — 사용자 체감 큼

### 1. 포스트 작성 화면 (`POST /posts`)
- **BE 상태**: ✅ 완성. multipart endpoint, 이미지 최대 10장, caption 2000자, recipeId 선택 태그
- **FE 현재**: Feed 탭 우측 상단 + 버튼이 `/_debug`로 가짜 라우팅
- **필요한 디자인 요소**:
  - 이미지 선택/카메라 (1~10장 그리드)
  - caption 입력 (멀티라인 텍스트 영역, 2000자 카운터)
  - 레시피 태그 검색/선택 모달 (선택)
  - 타입(PHOTO/TEXT) 결정 — 이미지 0장이면 TEXT 자동?
  - 등록 버튼 + 진행 중 표시
- **영향**: 현재 피드가 빈 이유 = 사용자가 직접 글 못 올림. **이게 없으면 커뮤니티 기능 사실상 죽음.**

### 2. 알림 화면 (`GET /notifications`)
- **BE 상태**: ✅ 완성 + 백그라운드 스케줄러 동작 중
  - 자동 생성 타입: `EXPIRY_WARNING`(D-14~D-8 주황), `EXPIRY_DANGER`(D-7~ 빨강), `POST_LIKE`, `POST_COMMENT`, `COMMENT_REPLY`
  - endpoint 4개: list, unread-count, mark-read, mark-all-read
  - 알림에 `link` 정보 포함 (`{ screen: 'INVENTORY_DETAIL' | 'POST_DETAIL', targetId }`)
- **FE 현재**: 화면 자체 없음, 라우트 없음
- **필요한 디자인 요소**:
  - 알림 리스트 화면 (`app/notifications.tsx` 신규)
  - 알림 아이콘 (헤더 어디?) + unread 뱃지
  - 타입별 아이콘 + 메시지 레이아웃
  - 빈 상태
  - 시간 표시 ("2시간 전")
  - 폴링/refresh 정책 (앱 포그라운드 진입 시 unread-count)
- **영향**: 사용자가 좋아요/댓글/유통기한 임박 알림 못 받음 → 재방문율 낮음

### 3. 보틀 수정/삭제 메뉴 (`PUT /inventory/{id}`, `DELETE /inventory/{id}`)
- **BE 상태**: ✅ 완성
- **FE 현재**: `bottle/[id]` 우측 상단 "..." 버튼이 `/_debug`로 가짜 라우팅
- **필요한 디자인 요소**:
  - "..." 누르면 액션 시트 (수정 / 삭제 / 취소)
  - 수정 화면: `bottle/new`와 같은 폼이지만 기존값으로 채워짐 (또는 `bottle/[id]/edit`)
  - 삭제 확인 다이얼로그
- **영향**: 잘못 등록한 보틀 못 고침 → UX 기본 가능 결손

## 🟨 우선순위 중간 — UX 개선

### 4. 검색 화면 (`GET /recipes?keyword=...`)
- **BE 상태**: ✅ 완성 (recipes 검색만. inventory 검색은 미지원)
- **FE 현재**: Recipe Book 우측 상단 검색 아이콘이 `/_debug`로 가짜
- **필요한 디자인 요소**:
  - 검색 입력창 화면
  - 최근 검색어 (선택)
  - 결과 리스트
  - 빈 결과 상태
- **영향**: 추천 외 적극 탐색 불가. CLAUDE.md §14에도 보류 항목으로 명시됨

### 5. 보틀 상세 화면의 새 필드 UI 정교화
- **BE 상태**: ✅ 4개 필드 추가 완료 (`tastingNotes`, `purchasedAt`, `purchasePlace`, `origin`)
- **FE 현재 (작업 18-2)**:
  - `bottle/[id]`: 표시는 들어가있음 (테이스팅 노트 칩, 구매일 타임라인, 구매처 행, 원산지 eyebrow)
  - `bottle/new`: 단순 텍스트 입력 4개 추가됨 (디자인 없어서 임시)
- **필요한 디자인 요소** (있으면 좋음):
  - `bottle/new`: 테이스팅 노트 → 칩 입력 컴포넌트 (쉼표 자동 변환)
  - `bottle/new`: 구매일 → 캘린더/날짜 피커 (현재는 `2026-05-18` 문자열 직접 입력)
  - `bottle/new`: 원산지 → 자동완성 (선택)
- **영향**: 입력 UX 어색함. 기능은 동작.

### 6. 레시피 별점 UI (`POST /recipes/{id}/ratings`)
- **BE 상태**: ✅ 별점 endpoint 4개 (upsert, getAll, getMy, delete) — Notion 명세에는 안 보이지만 코드에 있음
- **FE 현재**: `recipe/[id]` 하트 버튼이 로컬 state 토글만 (저장 안 됨)
- **필요한 디자인 요소**:
  - 별점 입력 UI (1~5 별 또는 다른 방식)
  - 별점 코멘트 입력 (선택)
  - 평균 별점 + 본인 별점 표시
- **영향**: 추천 알고리즘 정밀도, 사용자 데이터 자산

### 7. 마이프로필 화면 정교화
- **BE 상태**: ✅ `/users/me/profile` 병합 endpoint (profile + stats + 게시물 9개)
- **FE 현재 (작업 18-2)**:
  - 통계 3개 (POSTS / LIKES / RECIPES) 실제 데이터 연결됨 ✓
  - 선반 보틀 실제 인벤토리 연결됨 ✓
  - 바이오 텍스트는 여전히 mock ("위스키 3년차 · 스모키한 한 잔을 사랑하는 사람...")
  - 팔로우/메시지 버튼 `/_debug` 가짜
  - POSTS/RECIPES/SAVED 탭 영역 — `/posts/my` 호출해서 게시물 그리드 표시 가능하지만 디자인 없어서 빈 메시지
- **필요한 디자인 요소**:
  - 바이오 편집 화면 (BE는 컬럼 없음 — 필요하면 같이 요청)
  - 게시물 그리드 (3xN 썸네일)
  - "팔로우" 시스템 — BE에 없음. 디자인 필요 시 BE팀 요청 같이
- **영향**: 자기 활동 확인 가능. 현재 통계만 살아있음.

## 🟩 우선순위 낮음 — 후순위

### 8. AR 카메라 화면 (`POST /ar/sessions`)
- **BE 상태**: ✅ endpoint 완성. 단 명세상 **Unity 클라이언트가 AR 인식 담당** → RN은 결과만 업로드
- **FE 현재**: AR 탭 "준비 중" placeholder
- **필요한 디자인 요소** (만약 RN에서 직접 한다면):
  - AR 가이드 시작 화면
  - 카메라 뷰
  - 완료 후 사진 + 별점 + 캡션 입력
- **메모**: Unity로 가는 거면 RN은 placeholder 유지. MVP는 placeholder로 충분.

### 9. 빈 상태 / 로딩 / 에러 / 네트워크 끊김
- **FE 현재**: 일부 화면만 로딩 스피너 + 빈 상태 텍스트 (My Bar, Recipes, Bottle 등)
- **필요한 디자인 요소**:
  - 통일된 로딩 스피너 컴포넌트
  - 빈 상태 일러스트 + 메시지
  - 에러 토스트
  - 네트워크 끊김 배너
- **영향**: 완성도. 현재는 그냥 텍스트 1줄 + ActivityIndicator

### 10. 설정 화면
- **BE 상태**: 부분적 (logout, password reset). 설정 항목 자체는 별도 endpoint 없음
- **FE 현재**: 화면 자체 없음 (로그아웃은 profile 탭에 있음)
- **필요한 디자인 요소**:
  - 계정 관리 (이메일 변경, 비번 변경)
  - 알림 설정 (BE 미구현)
  - 약관/정책 보기
  - 회원 탈퇴 (BE 미구현)
- **영향**: 사용자 정책 대응. MVP 출시 전엔 약관/탈퇴는 필수일 수 있음 (법적 이슈)

### 11. 답글 작성 입력창 (`POST /posts/{id}/comments/{commentId}/replies`)
- **BE 상태**: ✅ 완성 (1-depth 대댓글까지만)
- **FE 현재**: `post/[id]`에 댓글의 "답글 N개"는 표시되지만 답글 추가 입력 불가
- **필요한 디자인 요소**:
  - 댓글 옆 "답글" 버튼
  - 답글 입력 UI (현재 댓글 입력바를 재사용? 모달?)
- **영향**: 깊이 있는 토론 불가

### 12. 댓글/포스트 삭제 (`DELETE` 시리즈)
- **BE 상태**: ✅ 완성
- **FE 현재**: 모든 list/detail 화면에 본인 글/댓글 삭제 UI 없음
- **필요한 디자인 요소**:
  - 본인 콘텐츠 옆 "..." 또는 길게 눌러서 삭제
  - 삭제 확인 다이얼로그
- **영향**: 잘못 올린 글 못 지움

### 13. 라벨 OCR 스캔 (`POST /inventory/scan`)
- **BE 상태**: ⚠️ stub (항상 매칭 실패 반환). OCR 라이브러리 미통합
- **FE 현재**: `bottle/new` 상단 "라벨 스캔으로 자동 등록" 카드는 클릭 안 됨
- **필요한 디자인 요소**: 카메라 스캐너 화면
- **메모**: BE도 실제 구현 필요. MVP 이후로.

### 14. 인증 가드 / 자동 라우팅
- **FE 현재**: `/_debug` 직진 (TEMP-DEBUG). 인증 가드 없음.
- **필요한 디자인 요소**: 디자인보다는 **흐름 정책** 결정 — 미로그인 시 어디로?
- **영향**: 배포 전 필수. `/_debug` 제거하고 `useAuthStore.isAuthenticated`로 분기 복원

## 🎯 디자이너에게 가져갈 우선순위 추천

만약 디자이너와 1회 미팅으로 결정해야 한다면:
1. **포스트 작성 화면** — 가장 큰 가치 (커뮤니티 살림)
2. **알림 화면** — 두 번째로 큰 가치 (재방문)
3. **보틀 수정/삭제 액션 시트** — 30분 디자인으로 큰 UX 결손 메움

위 3개가 디자인되면 FE 작업 가능 시간 **3~5일치** 풀림.
